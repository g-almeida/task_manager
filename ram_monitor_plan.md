# RAM Monitor Plan (Linux)

## Objective
Create a Python-based real-time RAM monitor that reads system data directly from `/proc/meminfo` to understand the underlying Linux mechanism.

## Understanding /proc/meminfo
On Linux systems, the `/proc` directory is a "pseudo-filesystem" (procfs). It doesn't contain real files but serves as an interface to kernel data structures.

- **What is `/proc/meminfo`?**: It is a dynamic text file generated by the kernel that reports the current state of the system's memory.
- **Key Fields**:
    - `MemTotal`: Total usable RAM (excluding the kernel's reserved binary code).
    - `MemFree`: The sum of low-free and high-free memory that is currently completely unused.
    - `MemAvailable`: An estimate of how much memory is available for starting new applications without swapping. This is usually more useful than `MemFree` because it accounts for cache/buffers that can be reclaimed.
    - `Buffers/Cached`: Memory used for disk caching, which the kernel can quickly free if applications need it.

## How the Script Reads Data
The implementation follows these logical steps to extract data:
1. **Open & Read**: The script opens `/proc/meminfo` like a standard text file.
2. **Line-by-Line Parsing**: It iterates through each line (e.g., `MemTotal: 8147548 kB`).
3. **Splitting & Cleaning**:
    - It splits the string by the colon `:` to separate the label from the value.
    - It strips whitespace and splits the value part to handle the "kB" suffix.
4. **Unit Conversion**: The kernel reports in kilobytes (kB). The script divides these values by `1024` to provide a more readable output in Megabytes (MB).
5. **Logic**: `Used Memory` is calculated as `MemTotal - MemAvailable`. This is the most accurate "modern" way to show how much RAM is actually being consumed by processes.

## Key Files & Context
- `/proc/meminfo`: The kernel interface for memory statistics.
- `ram_monitor.py`: The main script to be created.

## Implementation Steps
1.  **Parse /proc/meminfo**: Create a function to read `/proc/meminfo` and extract `MemTotal`, `MemAvailable`, and optionally `Buffers` and `Cached`.
2.  **Calculate Usage**: Calculate used memory (Total - Available) and percentage.
3.  **Real-time Loop**: Use a `while True` loop with `time.sleep(1)` to refresh the data every second.
4.  **Display Formatting**: Use ANSI escape codes or simple print statements to clear the console and show a clean output (Total, Used, Available, Percentage).
